# System Design: Enterprise RAG

## Why this lesson

Practice designing a robust RAG system for interviews.

## Outcomes

- ✅ High-level architecture with ingestion, retrieval, generation, eval
- ✅ Scaling and security considerations
- ✅ Trade-offs discussed

**Time**: ~30 minutes

**Tasks
1) Draw an enterprise RAG design
2) Address scaling, caching, eval, security
3) Prepare trade-off talking points

## Deliverable

- One-page RAG design diagram/notes

## Checkpoint

Can you explain how you’d handle stale documents and prompt injection?
